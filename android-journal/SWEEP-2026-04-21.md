# Full Sweep — The Claude Journal (2026-04-21)

## Ship Readiness: NOT READY

### Core Diagnosis

The codebase contains two parallel architectures — a pragmatic production layer that works, and a category-theory pattern library that is mathematically correct but never wired in. Every one of the 6 algebraic modules (Result, Validation, EventBus, CommandHistory, Mediator, SearchStrategy) is dead code. The security gaps in the production layer are exactly the gaps these patterns were designed to prevent.

## Top 5 Actions (highest impact, cross-cutting)

1. **Fix access control on unpublished papers** (blocks shipping) — download route, addNote, toggleFavourite, markAsRead all serve/accept data for unpublished papers. Paper IDs are sequential (2026-001, 2026-002...) making enumeration trivial. Add `status: 'published'` filter everywhere, with editorial bypass.

2. **Fix review data field injection** (blocks shipping) — `submitReview` passes entire `ReviewData` to `prisma.review.update({ data })`. Attacker can set `visible: true` to bypass editorial workflow. Destructure and whitelist fields.

3. **Wire or remove the dead pattern library** (highest-impact debt) — ~750 lines of library code and ~1200 lines of tests exercise code that never runs in production. Triage:
   - **Wire now:** Result monad, middleware stacks, Auth adapter, Validation combinators
   - **Keep for V2:** EventBus, CommandHistory
   - **Delete:** `search/tsvector.ts` (duplicate), SubmissionMediator (premature for V1)

4. **Add integration tests for security-critical paths** (zero coverage today) — auth bypass, unpublished paper access, review field injection, path traversal, search injection.

5. **Wire pino logging with correlation IDs** — pino is a dependency but never imported. `withTrace` creates correlation IDs that are computed and discarded.

## Consolidated Remediation Plan

### Phase 0: Ship Blockers (1-2 days)

1. Add `status: 'published'` filter to download route, `addNote`, `toggleFavourite`, `markAsRead` (with editorial bypass)
2. Whitelist fields in `submitReview` — never pass raw user data to `prisma.update()`
3. Add optimistic locking to `transitionPaper()` — `WHERE status = $expected` in update
4. Fix paper ID generation — use `SELECT ... FOR UPDATE` or PostgreSQL sequence
5. Fix buggy async filter in `role-lattice.test.ts`

### Phase 1: Wire the Patterns (3-5 days)

6. Replace `{ success, error }` returns in all server actions with `Result<T, E>`
7. Use Validation combinators for input validation in `submitPaper`, `submitReview`, `addNote`
8. Use middleware stacks (`authRoute`, `editorRoute`) in API routes instead of inline `getSession()` calls
9. Use `GitHubAuthAdapter` in the OAuth callback route
10. Delete `lib/search/tsvector.ts` and `lib/submission/mediator.ts`
11. Update DESIGN_PATTERNS.md to reflect actual wiring state

### Phase 2: Production Hardening (first sprint)

12. Wire pino with correlation IDs from `withTrace`
13. Add Next.js `middleware.ts` with security headers and rate limiting
14. Add input length limits on all text fields
15. Remove `uuid` dependency
16. Add health check endpoint (`/api/health`)
17. Document `search_vector` migration in Prisma schema comments

### Phase 3: Test Coverage (first sprint)

18. Integration tests: auth bypass, access control, field injection, path traversal, search injection
19. Contract tests: response shapes, status codes, content-types for all API routes
20. Server action tests: `submitPaper`, `updatePaperStatus`, `submitReview` with valid/invalid inputs
21. Add file cleanup tests for partial submission failure
